## Models

Models are the core interfaces through which we interact with AI models. For example, if we are building a chatbot, the model could be OpenAI, Gemini, or any other provider — and the beauty of LangChain is that **switching between models requires minimal code changes**.

**Without LangChain** — each provider has its own SDK and API structure:

```python
# Using OpenAI directly
from openai import OpenAI

client = OpenAI(api_key="...")
response = client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "What is LangChain?"}]
)
print(response.choices[0].message.content)
```

```python
# Using Gemini directly
import google.generativeai as genai

genai.configure(api_key="...")
model = genai.GenerativeModel("gemini-pro")
response = model.generate_content("What is LangChain?")
print(response.text)
```

Switching between these requires rewriting the entire integration logic.

**With LangChain** — switching models is a one-line change:

```python
from langchain_openai import ChatOpenAI
from langchain_google_genai import ChatGoogleGenerativeAI

# Use OpenAI
llm = ChatOpenAI(model="gpt-4", api_key="...")

# Switch to Gemini — just swap this one line
# llm = ChatGoogleGenerativeAI(model="gemini-pro", google_api_key="...")

response = llm.invoke("What is LangChain?")
print(response.content)
```

The rest of the code (chains, prompts, output parsers) stays **exactly the same**.

## Prompts

Prompts are the instructions passed to an LLM. The better we frame the prompts, the better the LLM response will be.

### Role-based prompts

We assign the LLM a particular role using a **system** prompt, followed by a user query. Based on the assigned role, the LLM tailors its response accordingly.

```python
from langchain_openai import ChatOpenAI
from langchain_core.messages import SystemMessage, HumanMessage

llm = ChatOpenAI(model="gpt-4")

messages = [
    SystemMessage(content="You are a senior Python developer. Explain concepts with code examples."),
    HumanMessage(content="What are decorators?")
]

response = llm.invoke(messages)
print(response.content)
```

By changing just the `SystemMessage`, the same LLM behaves as a different "expert" — a data scientist, a marketing copywriter, a legal advisor, etc.

### Few-shot prompts

We provide a few examples within the conversation. The LLM learns from these examples **how** to respond, and then applies that pattern to answer new queries.

```python
from langchain_core.prompts import FewShotPromptTemplate, PromptTemplate

# Teach the LLM to classify customer support messages
examples = [
    {"message": "My order hasn't arrived yet", "category": "Shipping"},
    {"message": "I was charged twice for the same item", "category": "Billing"},
    {"message": "The product broke after one day", "category": "Product Quality"},
]

example_prompt = PromptTemplate(
    input_variables=["message", "category"],
    template="Message: {message}\nCategory: {category}"
)

few_shot_prompt = FewShotPromptTemplate(
    examples=examples,
    example_prompt=example_prompt,
    prefix="Classify the following customer message into a support category.",
    suffix="Message: {input}\nCategory:",
    input_variables=["input"],
)

print(few_shot_prompt.format(input="I want to return this product"))
# Output:
# Classify the following customer message into a support category.
#
# Message: My order hasn't arrived yet
# Category: Shipping
#
# Message: I was charged twice for the same item
# Category: Billing
#
# Message: The product broke after one day
# Category: Product Quality
#
# Message: I want to return this product
# Category:
```

The LLM learns the classification pattern from the examples and responds with `Returns`.

### Dynamic & reusable prompts

We prepare a generalized prompt template with **variables** (placeholders). These variables are replaced based on the user's query at runtime, and the LLM responds accordingly.

```python
from langchain_core.prompts import PromptTemplate

# Define a reusable template
template = PromptTemplate(
    input_variables=["language", "topic"],
    template="Explain {topic} in {language} with a simple example."
)

# Reuse the same template for different inputs
prompt1 = template.format(language="Python", topic="list comprehension")
prompt2 = template.format(language="JavaScript", topic="promises")
prompt3 = template.format(language="Go", topic="goroutines")

print(prompt1)
# Output: Explain list comprehension in Python with a simple example.
```

One template, many use cases — no need to write separate prompts for each combination.

## Chains

Chains are sequences of components executed together. The output of one step is automatically fed as input to the next — no boilerplate code needed for passing data between steps. For complex scenarios, chains can run in **parallel** as well.

**Sequential chain** — pipe a prompt through an LLM and then parse the output:

```python
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

llm = ChatOpenAI(model="gpt-4")

prompt = ChatPromptTemplate.from_template("Write a one-line joke about {topic}")

# Chain: prompt → LLM → parse output as string
chain = prompt | llm | StrOutputParser()

result = chain.invoke({"topic": "programming"})
print(result)
```

**Multi-step chain** — combine multiple operations:

```python
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

llm = ChatOpenAI(model="gpt-4")

# Step 1: Generate a blog title
title_prompt = ChatPromptTemplate.from_template("Suggest a blog title about {topic}")
title_chain = title_prompt | llm | StrOutputParser()

# Step 2: Write an outline based on the generated title
outline_prompt = ChatPromptTemplate.from_template("Write a 3-point outline for a blog titled: {title}")
outline_chain = outline_prompt | llm | StrOutputParser()

# Combine: output of step 1 feeds into step 2
full_chain = title_chain | (lambda title: {"title": title}) | outline_chain

result = full_chain.invoke({"topic": "AI in healthcare"})
print(result)
```

The `|` (pipe) operator is LangChain's way of chaining components — clean, readable, and easy to extend.

## Indexes

Indexes connect our application to **external knowledge sources** — PDFs, websites, databases, and more. They are the backbone of RAG (Retrieval-Augmented Generation) systems.

**Main components:**

| Component | Purpose |
|-----------|---------|
| **Document Loader** | Loads raw data from sources (PDF, CSV, web pages, Notion, etc.) |
| **Text Splitter** | Breaks large documents into smaller, manageable chunks |
| **Vector Store** | Stores chunk embeddings for fast similarity search (e.g., Pinecone, Chroma, FAISS) |
| **Retriever** | Fetches the most relevant chunks for a given query |

## Memory

LLM API calls are **stateless** — each call is independent and has no knowledge of previous conversations. Memory solves this by storing conversation history so the LLM can answer follow-up queries with full context.

### Without memory

```python
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4")

llm.invoke("My name is Alice.")
response = llm.invoke("What is my name?")
print(response.content)
# Output: I don't know your name. You haven't told me.
```

### With memory

```python
from langchain_openai import ChatOpenAI
from langchain.memory import ConversationBufferMemory
from langchain.chains import ConversationChain

llm = ChatOpenAI(model="gpt-4")
memory = ConversationBufferMemory()
conversation = ConversationChain(llm=llm, memory=memory)

conversation.predict(input="My name is Alice.")
response = conversation.predict(input="What is my name?")
print(response)
# Output: Your name is Alice.
```

### Types of memory

| Type | Description | When to use |
|------|-------------|-------------|
| **ConversationBufferMemory** | Stores the entire conversation history as-is | Short conversations where full context matters |
| **ConversationBufferWindowMemory** | Keeps only the last **N** interactions | Long chats where only recent context is relevant (saves tokens) |
| **SummarizerBasedMemory** | Summarizes older messages to keep memory compact | Very long conversations where key facts should be retained without token bloat |
| **Custom Memory** | User-defined memory logic (e.g., store only specific entities, facts, or metadata) | Domain-specific needs like tracking user preferences or extracted entities |

> 💡 **Tip:** In real-world applications, you'll often **combine multiple memory types** rather than picking just one. For example, a customer support chatbot might use `ConversationBufferWindowMemory` to retain the last few exchanges verbatim, `SummarizerBasedMemory` to keep a running summary of the older conversation, and `Custom Memory` to persistently track user-specific details (name, order ID, subscription tier). This hybrid approach keeps recent context sharp, older context compact, and key facts always available — all while controlling token usage.

**Example — combining memory types with `CombinedMemory`:**

```python
from langchain_openai import ChatOpenAI
from langchain.memory import (
    ConversationBufferWindowMemory,
    ConversationSummaryMemory,
    CombinedMemory,
)
from langchain.chains import ConversationChain
from langchain_core.prompts import PromptTemplate

llm = ChatOpenAI(model="gpt-4")

# 1. Keep the last 3 exchanges verbatim (recent context)
window_memory = ConversationBufferWindowMemory(
    k=3,
    memory_key="recent_history",
    input_key="input",
)

# 2. Keep a running summary of older conversation (compact long-term context)
summary_memory = ConversationSummaryMemory(
    llm=llm,
    memory_key="conversation_summary",
    input_key="input",
)

# 3. Combine both into a single memory object
combined_memory = CombinedMemory(memories=[window_memory, summary_memory])

# Custom prompt that uses both memory keys
prompt = PromptTemplate(
    input_variables=["conversation_summary", "recent_history", "input"],
    template="""You are a helpful customer support agent.

Summary of older conversation:
{conversation_summary}

Recent messages:
{recent_history}

User: {input}
Agent:"""
)

conversation = ConversationChain(
    llm=llm,
    memory=combined_memory,
    prompt=prompt,
)

conversation.predict(input="Hi, I'm Alice and my order ID is #12345.")
conversation.predict(input="It hasn't arrived yet.")
conversation.predict(input="Can you check the status?")
response = conversation.predict(input="What was my order ID again?")
print(response)
# The agent can still recall "#12345" — pulled from the summary or recent window.
```

You can plug in `Custom Memory` the same way to track structured facts (e.g., user profile, cart contents) alongside these two.

## Agents

In simple words, **AI agents** can be considered as an advanced version of a chatbot — one that has **reasoning capability** and **access to tools**. Instead of just answering queries like a normal chatbot, an agent can **take actions on your behalf** by deciding which tool to use and when.

### How agents work

An agent follows a loop of **Reason → Act → Observe**:

1. **Reason** — the LLM analyzes the user query and decides what to do
2. **Act** — it calls a tool (search, calculator, API, database, etc.)
3. **Observe** — it reads the tool's output and decides the next step
4. Repeat until the final answer is ready

### Example — an agent with tools

```python
from langchain_openai import ChatOpenAI
from langchain.agents import initialize_agent, Tool, AgentType
from langchain_community.tools import DuckDuckGoSearchRun

llm = ChatOpenAI(model="gpt-4")
search = DuckDuckGoSearchRun()

# Define tools the agent can use
tools = [
    Tool(
        name="Web Search",
        func=search.run,
        description="Useful for answering questions about current events or recent information."
    ),
    Tool(
        name="Calculator",
        func=lambda expr: str(eval(expr)),
        description="Useful for math calculations. Input should be a valid Python expression."
    ),
]

agent = initialize_agent(
    tools=tools,
    llm=llm,
    agent=AgentType.ZERO_SHOT_REACT_DESCRIPTION,
    verbose=True,
)

agent.run("What is the current population of Tokyo, and what is 15% of that number?")
# The agent will:
# 1. Use Web Search to find Tokyo's population
# 2. Use Calculator to compute 15% of it
# 3. Return the final answer
```

### Real-world agent use cases

| Use Case | What the agent does |
|----------|---------------------|
| **Travel booking assistant** | Search flights, check weather, book hotels via APIs |
| **Personal finance bot** | Fetch stock prices, calculate portfolio returns, send alerts |
| **DevOps assistant** | Query logs, restart services, create Jira tickets |
| **Research assistant** | Search the web, summarize papers, save notes to Notion |
| **Customer support agent** | Look up order status, process refunds, escalate tickets |

The key difference between a chatbot and an agent — a chatbot **tells** you what to do; an agent **does** it for you.
