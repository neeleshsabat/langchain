## Models

Generally, language models are classified into two categories — **LLMs** and **Chat Models**.

- **LLMs** are general-purpose models primarily used for **text generation**. They accept a plain string as input and return a string as output.
- **Chat Models** are specialized for **conversational tasks**. They accept a sequence of structured messages (system, user, assistant) and return a message in response.

### LLMs vs Chat Models

| Aspect | LLM | Chat Model |
|--------|-----|------------|
| **Input** | A single string (plain text) | A list of messages (`SystemMessage`, `HumanMessage`, `AIMessage`) |
| **Output** | A string | A message object (typically `AIMessage`) |
| **Use case** | Text completion, summarization, simple generation | Chatbots, multi-turn conversations, role-based interactions |
| **Roles support** | No concept of roles | Built-in roles (system / user / assistant) |
| **Conversation memory** | Must be manually appended to the prompt | Naturally fits message-based memory |
| **LangChain class** | `OpenAI`, `HuggingFaceHub`, etc. | `ChatOpenAI`, `ChatAnthropic`, `ChatGoogleGenerativeAI`, etc. |
| **Example providers** | GPT-3 (`text-davinci-003`), older completion APIs | GPT-4, Claude, Gemini, Llama-chat |
| **Recommended for** | Simple, one-shot text tasks | Most modern applications (chatbots, agents, RAG) |

**Quick code comparison:**

```python
# LLM — string in, string out
from langchain_openai import OpenAI

llm = OpenAI()
response = llm.invoke("Write a haiku about the sea.")
print(response)  # plain string
```

```python
# Chat Model — messages in, message out
from langchain_openai import ChatOpenAI
from langchain_core.messages import SystemMessage, HumanMessage

chat = ChatOpenAI(model="gpt-4")
response = chat.invoke([
    SystemMessage(content="You are a poet."),
    HumanMessage(content="Write a haiku about the sea."),
])
print(response.content)  # extracted from AIMessage
```

> 💡 **Tip:** For most modern applications, prefer **Chat Models** — they support roles, structured messages, and integrate seamlessly with LangChain's memory and agent abstractions. LLMs are mostly used today for simple, single-turn text tasks.
